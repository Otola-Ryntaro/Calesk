# 011_account_config_defensive_parsing

## 対応対象
- `src/calendar_client.py`

## 問題
- `accounts.json` 読み込み時に一部 `account['id']` 直接参照が残っており、破損データで `KeyError` を起こしうる。

## 修正方針（どう直すか）
- すべて `.get()` ベースで防御的に読み、必須値欠落エントリはスキップ。
- スキップ理由をログへ出力して復旧しやすくする。

## 実装タスク
- [ ] 直接キー参照箇所の洗い出しと置換
- [ ] 必須キー検証関数の追加（任意）
- [ ] 不正レコード混在時も処理継続する挙動へ統一

## 受け入れ条件
- [ ] 壊れた `accounts.json` でもアプリ全体が停止しない
- [ ] 正常レコードは継続利用できる

## テスト観点
